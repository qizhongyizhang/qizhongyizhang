写在前面：本文主要记录 数据资产管理 的相关基础内容

数据资产

数据资产：由企业拥有或控制的（包括以合法途径获取的企业外部数据资源），未来能够带来经济利益的，以物理或电子方式记录的数据资源等，如文档资料、电子数据等
数据资产管理：对数据资产进行管理，规划、控制数据及信息资产的业务活动，包括开发、执行和监督有关数据的计划、政策、项目等，确保数据资产保值、升值，贯穿数据整个生命周期

1. 数据资产特性

无消耗性：数据资产不会因为使用频率的增加而磨损、消耗，与传统无形资产类似
增值性：企业通过稳定发展，数据规模和数据维度会在原有的基础上不断积累，整体价值进一步提升
依附性：数据资产往往依附于相应的软件、硬件发挥作用和效应，不能独立发挥作用
价值易变性：数据资产时刻受到数据容量、数据时效程度、应用场景等因素的影响，其价值更易发生变化
战略性：一切数据业务化，一切业务数据化，具有战略性

2. 数据资产估值

2.1 估值内容

  1. 运营成本：包括采集、存储、计算和运维的费用
  2. 替换成本：在灾难性数据破坏事件或数据中断时，数据替换货恢复的成本
  3. 风险成本：潜在罚款、补救成本和诉讼费用的估价
    • 缺少必须数据
    • 保存不应留存数据
    • 存储数据不正确
  4. 使用价值：企业日常运行带来的价值，主要通过数据资产分类、使用频次、使用对象、使用效果和共享流通进行计量
  5. 市场价值：通过交易、售卖、兼并或收购时作为企业资产的价值，通过稀缺性、时效性进行衡量
  6. 洞察价值：从数据中发现、洞察商机获得的收入价值

2.2 估值方法

  • 非财务方法

    数据的内在价值模型：不考虑业务价值，仅从技术层面进行特征加权统计
    数据的商业价值模型：结合业务流程进行特征加权统计
    数据的绩效价值模型：衡量随时间推移对API绩效指标的影响

  • 财务方法

    数据的成本价值模型：衡量获取或替换丢失数据的价值成本
    数据的经济价值模型：衡量信息资产对企业收入的贡献
    数据的市场价值模型：衡量企业出售、出租和交换企业数据所产生的收入

2.3 价值评估理论

  • 层次分析法

    分析各因素关系，建立地接层次结构
    同一层级进行重要性判断，构建判断矩阵
    由判断矩阵计算比较元素的相对权重
    计算合成权重

  • 专家打分法（德尔菲法）

    按照课题组成专家小组
    收集专家关于相关问题的书面回答
    分析专家的回答，提出自己的预测
    将专家判断意见汇总，进行对比，再将所有专家意见返还，专家根据他人意见进行修改
    回收专家意见，再次汇总分发，直至所有专家不再更改
    对专家意见进行综合处理

2.4 评估指标

  • 数据内在价值——数据质量

    完整性：缺失字段或缺失记录（整个数据缺失记录）
    正确性：数据内容与客观实体特征是否一致
    一致性：同一实体同一属性值在不同的数据集中是否一致
    重复性：是否存在重复数据

  • 数据外在价值

    稀缺性：数据供给方数量的多寡及数据的供给丰富程度
    时效性：数据时间特性对应用的满足程度
    多维性：描述数据集的多寡程度
    场景经济性：在具体场景中数据集的经济价值

3. 数据资产目录

包含内容：业务术语表、数据元素定义、数据血缘、数据质量标准、数据安全标准等

3.1 编制方法

系统视角：以系统目前的核心系统为主，将系统功能模块分类，按照系统数据主题、实体定义信息、实体分类信息、数据相关方信息、技术信息构建数据资产目录
主题域视角：按照战略发展、业务运营、管理支持构建系统统一的数据域主题或以及主题，按照实体定义信息、实体分类信息、数据相关方信息、技术信息构建数据资产目录

3.2 能力评估

  • 能力内容

    发现能力：所有可用数据资产进行清洗全面概述的能力
    理解能力：新增、修改数据资产目录和自我丰富的能力
    信任能力：支持用户评估数据的可靠性和质量
    协作能力：支持用户进行任务共享并鼓励用户创建和修订描述
    治理能力：支持满足企业标准的能力

  • 能力级别

    初始级：没有努力集中编制数据资产目录，无法集中获得大部分有价值信息
    主动管理：存在数据资产列表和文档，可使用搜索功能，更新缓慢
    基于工具：数据资产目录由专业团队管理，能助力自动捕获元数据，使用更轻松
    优化级：数据资产目录做了样本数据条目和统计信息，包含自动捕获的血缘信息，使数据资产目录可信度更高
    自动化级：可向用户推荐数据资产可自动生成数据质量和数据资产价值评估结果
