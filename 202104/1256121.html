Linux-常见指令

Linux
常见操作指令

写在前面：本文主要记录Linux操作系统的常见操作指令

命令格式：command [-option] [parmeter]

1. 指定运行级别

Init 运行级别

  • 0：关机
  • 1：单用户（找回丢失密码）
  • 2：多用户无网络服务
  • 3：多用户有网络服务，常用
  • 4：保留的运行级别
  • 5：图形界面
  • 6：重启
    * 配置文件：/etc/inittab的 id:5:initdefault
  • init 0/1/2/3/5/6

2. 帮助指令

  • man 指令，获得帮助信息（the on-line reference manuals）

例：man ls

  • help 指令，获得shell内置命令的帮助信息

例：help cd, cd –help 皆可

3. 文件目录

  • pwd：显示当前目录的绝对路径

例：pwd

  • ls [选项] [目录或文件]：显示指定目录下的文件和目录（list directory contents），默认为当前目录

选项：

-a：显示当前目录下的所有文件和目录，包括隐藏内容，隐藏文件以 . 开头
-l：以列表形式显示文件详细信息信息
-h：配合 -l 以人性化的方式显示文件大小

例：ls -al ：以列表形式显示当前目录下包括隐藏内容的所有文件和目录
例：ls -l -h ；ls -lh ；ls -hl ；不关注顺序

通配符：

*：匹配任意多个字符，例：ls 1* 显示以 1 开头的文件
?：匹配任意一个字符，例：ls 1?.txt 显示以 1 开头且文件名长度为 2 的 txt 文件
[]：匹配字符组，例：ls [1-9].txt 显示文件名为 1-9 的 txt 文件，ls [1, d, _].txt

  • cd [选项]：切换目录（change dictionary）

例：cd .. ：返回上一级目录
例：cd . ：切换至当前目录
例：cd ~：返回当前用户的home目录/主目录，也可以使用：cd ；后面不跟内容
例：cd -：在最近两次工作目录间切换
例：cd 路径 ：前往指定目录，可用绝对路径(cd /home/xxx)和相对路径(cd ./xxx)

  • mkdir [选项] 目录名：创建目录（make directories）

选项：-p，创建多级目录，默认一次只能创建一级目录

例：mkdir -p /home/animal/dog ：在home目录下创建animal，再在animal下创建dog，若之前有animal目录则不需要-p选项

  • rmdir [选项] 目录名：删除空目录（remove empty directories）

若要删除非空目录，应使用：rm -r 目录名

  • touch 文件名：创建空文件，可一次创建多个文件；如果文件已存在，就修改文件的末次修改时间

例：touch hello.txt helloword.txt
例：touch /home/user/xxx.txt ；允许级联创建

  • rm [选项] 目录名或文件名：删除文件或目录（remove），默认有删除提示；删除后不能恢复；可以使用通配符

    选项：

    -r：递归删除整个文件夹
    -f：强制删除不提示
    -rf：强制递归删除整个文件夹不提示（慎用！！！）

    例：rm -r * 删除当前文件夹内的所有文件和目录

  • tree [选项] 目录名：以树状图列出文件目录结构；省略则输出当前文件夹的目录结构

    选项：

    -d 只显示目录

    例：tree -d ~，显示根目录下的目录树

  • cp [选项] source(数据源) dest(目标位置)：复制文件到指定目录（copy）

    选项：

    -r 递归复制整个文件夹
    -i 覆盖文件前给予提示

    例：cp -r test1/ test2/：将test1整个复制到test2里面

    SCP：远程CP，-P 指定端口

  • mv 移动或重命名文件或目录（move）

    移动

    文件：mv file_name dictionary_name
    文件夹：mv dictionary_name dictionary_name

    重命名

    文件：mv old_file_name new_file_name（如果存在同名，会覆盖；支持 -i 覆盖文件前给予提示）
    文件夹：mv old_dictionary_name new_dictionary_name

  • cat [选项] 目录名：以只读的方式打开文件；将文件一次性全部显示，适用于文件内容较少的文件

选项

-n：显示行号，包含空行
-b：显示行号，不包含空行

  • more [选项(-dflpcsu)] [-num] [+linenum | +/pattern] 文件名：以全屏的方式按页显示文件内容，加载完全部内容，再进行显示，按空格键进行翻页

选项

  • -d：提示使用者，在画面下方显示 [Press space to continue, ‘q’ to quit.] ，如果使用者按错键，则会显示 [Press ‘h’ for instructions.] 而不是 ‘哔’ 声
  • -l：取消遇见特殊字符，默认是将^L(表单提要)视为特殊字符，在包含表单提要的任何行之后暂停，使用-l会避免上述情况
  • -f：计算行数时，以实际上的行数，而非自动换行过后的行数(有些单行字数太长的会被扩展为两行或两行以上)
  • -p：不以卷动的方式显示每一页，而是先清除萤幕后再显示内容
  • -c：跟 -p 相似，不同的是先显示内容再清除其他旧资料
  • -s：当遇到有连续两行以上的空白行，就代换为一行的空白行
  • -u：不显示下引号 (根据环境变数 TERM 指定的 terminal 而有所不同)
  • -num：设置一次显示的行数
  • +linenum：开始显示的行号
  • +/pattern：在每个文档显示前搜寻该字串(pattern)，然后从该字串之后开始显示

快捷键

  • 空格键：向下翻页
  • Enter：下翻一行
  • q：退出more指令
  • Ctrl + F：向下滚动一屏
  • Ctrl + B：向上滚动一屏
  • =：显示当前行号
  • :f：输出文件名和当前行号
  • less [选项] 文件名：功能同more，支持各种显示终端，根据显示需要动态加载内容，对显示大型文件有较高的效率

快捷键

  • 空格键/PageDown：向下翻页
  • PageUp：向上翻页
  • Enter：下翻一行
    /字符串：向下查找字符串内容：n-向下查找；N-向上查找
    ?字符串：向上查找字符串内容：n-向上查找；N-向下查找
  • q：退出less指令
  • head [-n] 文件名：显示文件开头部分内容，默认显示10行，n为正整数
  • tail [选项] 文件名：显示文件结尾部分内容，默认显示10行

-n：设置显示的文件内容行数，n为正整数
-f：实时监控文件的更新情况

  • ln -s [原文件或目录] [连接名]：软连接，符号链接，类似于Windows的快捷方式，存放连接其他文件的路径；使用绝对路径，-s 表示建立软连接，省略表示建立硬链接（删除源文件，硬链接仍可以访问）
  • history [n]：查看已经执行过的指令，也可以执行历史指令，n为正整数，显示最近执行的n条指令，使用 !n，可以执行历史编号为n的指令

4. 时间日期

  • date [选项] [+ 显示格式]：显示当前时间，注意date与+之间有一个空格

显示格式：

%Y：显示当前年份
%m：显示当前月份
%w：显示当前星期
%d：显示当前日期
%Y%%d%H%M%S：显示当前年月日时分秒，可以在中间加入分隔符，date ‘+%Y年%m月%d日 %H:%M:%S’，需要用引号包裹
选项
-s：设定当前系统时间：date -s ‘yyyy-mm-dd HH:MM:ss’

  • cal [时间]：默认显示当前月的日历，时间为年份，加上时间显示指定年份一整年的日历；cal -y

5. 搜索查找

  • find [范围] [选项]：从指定目录下递归遍历各个子目录，将满足条件的内容显示在终端

选项：

-name ：按照文件名进行查找，例：find / -name *1.txt，查找文件名以 1 结尾的 txt 文件
-user：按照文件拥有者进行查找
-size [+/-]Size：按照文件大小进行查找，+表示大于，-表示小于，不写表示等于Size，Size表示文件大小，100k、20M

  • locate 文件名称：根据locate数据库快速定位文件路径，locate存储系统中所有文件名称及路径，需要及时更新，第一次执行前，需要使用updatedb指令创建locate数据库
  • grep [选项] 查找内容 源文件：过滤查找，管道符 | ，表示将前一个命令的处理结果输出传递给后面的指令处理

选项：

-n：显示行号
-i：忽略大小写
-v：返回不包含指定内容的行，包含空行

例：grep -niv abc file_name ；返回 file_name 内包含 abc 的行，显示行号，同时忽略大小写

模式查找：也成为正则表达式，可配合选项进行操作

^a：查找以 a 开头的行
a$：查找以 a 结束的行

例：grep -n ^a file_name

6. 打包压缩

  • tar [选项] 文件名：打包指令，不进行压缩，打包文件为 .tar 后缀

选项：

-c：创建打包文件，生成档案文件
-f：指定档案文件名称，必须放在最后
-v：列出打包或解包的详细过程，显示进度
-x：解开打包文件
-z：调用 gzip ，实现压缩和解压缩命令；打包压缩文件为 .tar.gz 后缀（用tar进行打包，gzip进行压缩）

-j：调用 bzip2 ，实现压缩和解压缩命令；打包压缩文件为 .tar.bz2 后缀

例：tar -cvf 1_tar.tar 1* ，将当前目录下的以 1 开头的文件打包为1_tar.tar；解包 tar -xvf 1_tar.tar

例：tar -zcvf 1_tar.tar.gz 1* ，将当前目录下的以 1 开头的文件打包并压缩为1_tar.tar.gz；解压缩包 tar -zxvf 1_tar.tar.gz [-C] 指定解压缩的目录（目录必须存在）

  • gzip/gunzip 文件名：gzip压缩文件，不保留源文件；gunzip解压文件，压缩文件为gz后缀

  • bzip2：与 tar 命令结合实现打包压缩

例：tar -jcvf 1_tar.tar.bz2 1*

  • zip/unzip [选项] 文件名：压缩文件为zip后缀

选项：

-r：递归压缩，压缩文件目录，例：zip -r 文件名 目录/
-d 目录：指定解压目录，例：unzip -d /目录 文件名

7. 定时机制

crontab [选项]

选项

-e：开始任务
-r：删除所有任务
-l：显示当前的cron任务
service crond start ：重启任务调度

例：每分钟将etc的目录内容写入到tmp的to.txt文件中
1. cron -e
2. 输入：\*/1 * * * * ls -l /etc/ >> /tmp/to.txt
3. 保存退出后生效

corn表达式

第一个*：一小时中的第几分钟，取0-59
第二个*：一天的第几小时，取0-23
第三个*：一月中的第几天，取1-31
第四个*：一年中的低级月，取1-12
第五个*：一周中的星期几，取0-7（0、7代表星期天）
可用*代表任何时间，用,指定多个不连续的时间，例：1,3,6，表示仅1，3，6执行，用-表示范围，例：1-6，表示1至6都执行，用/n表示多久执行一次，例：*/1 * * * *，每分钟执行一次

8. 磁盘管理

  • df [选项] ：disk free，查询系统整体磁盘剩余空间

-l：以列表形式显示信息
-h：按照用户可理解的形式显示，带计量单位

  • du [选项]：disk useage，查询指定目录的磁盘占用情况

-s：指定目录占用大小汇总
-h：按照用户可理解的形式显示，带计量单位
-a：含文件
-c：列出明细的同时，增加汇总值
–max-depth = 1：子目录的深度
例：du -ach –max-depth = 1 /opt，统计/opt目录的磁盘占用情况，深度为1

  • 常用

ls -l /home | grep “^-“ | wc -l，统计home文件夹下文件的个数，“^-”表示以-开头，wc表示统计
ls -lR /home | grep “^-“ | wc -l，统计home文件夹及子文件下文件的个数
tree /home，以树状图显示home文件夹

9. 进程管理

9.1 基本介绍

  • 每个执行的程序（代码）都成为一个进程，每一个进程都分配一个ID号
  • 每一个进程都会对应一个父进程，父进程可以复制多个子进程，Linux最大的进程是init
  • 每一个进程都能以前台和后台两种方式存在，前台进程就是用户在米钱屏幕上可以进行操作，后台进程则实际在操作，但屏幕上无法显示，一般采用后台方式执行
  • 一般系统的服务都是以后台进程的方式存在，会常驻在系统中，知道关机结束

9.2 进程管理-进程

  • ps [选项]：process status查看当前系统中正在执行的进程和其执行情况

选项

-a：当前终端的所有进程信息
-f：全格式显示
-u：显示进程详细状态信息
-x：显示后台进程
进程信息
USER：用户名
PID：进程识别号
%CPU：此进程占用CPU的百分比情况
%MEM：此进程占用内存的百分比情况
VSZ：此进程的虚拟内存占用情况
RSS：此进程的物理内存占用情况
TTY：终端机号
STAT：此进程的运行状态：S-休眠，s-表示此进程是会话的先导进程，R-运行，D-短期等待，Z-僵死进程，T-被跟踪或被停止的进程，N-表示此进程比普通进程的优先级更低
START：此进程的启动时间
TIME：此进程占用CPU的总时间
COMMAND：进程执行时的命令和参数

查看父进程：ps -ef | more，显示的PPID为父进程进程号
查看指定进程的父进程：ps -ef | sshd，查看sshd的父进程

  • top [选项]：显示正在执行的进程，top会在运行一段时间可以更新正在运行的进程

    选项

    -d n：指定n秒后自动更新，默认三秒
    -i：不显示任何闲置或僵死进程
    -p：通过指定进程ID进监控进程状态
    交互说明
    P：以CPU使用规律排序，默认选项
    M：以内存使用率排序
    N：以PID排序
    q：退出top指令

    使用

    监控指定用户：先输入top，回车，在输入u，最后输入用户名
    中止指定服务：先输入top，回车，在输入k，最后输入进程号

  • pstree [选项]：查看进程树

-p：显示进程的PID
-u：显示进程的所属用户

  • kill [选项] 进程号

选项：-9 强制进程立即终止

  • killall 进程名，支持通配符

终止：当进程需要停止或消耗了大量的系统资源时执行

  • 常见进程

sshd：远程登陆
bash：终端命令行窗口

9.3 进程管理-服务

9.3.1 基本介绍

服务（Service）的本质是后台运行的进程，通常会监听某一个端口，等待其他程序的请求，又叫守护进程，每一个服务都会在0-6的Linux运行级别下通过chkconfig设置是否自启动

service 服务名 start|stop|restart|reload|status

CentOS 7.0以后使用systemctl，不再使用service

重启防火墙：service iptables restart
查看防火墙状态：service iptablesstatus

关闭或启用防火墙后立即生效，可用telnet ip 端口进行测试
设置仅当次有效，系统重启后回到原有设置，可用chkconfig设置永久生效
查看服务
setup，进入系统服务，加入_*_号的服务均会自启动，可以直接修改
查看/etc/init.d目录下的内容
常见服务
防火墙：iptables

chkconfig；给每个服务指定各个运行级别是否自启动

查看所有服务：chkconfig –list
查看指定服务：

chkconfig –list | grep 服务名
chkconfig 服务名 –list
修改指定服务指定运行级别的自启动状态：chkconfig –level 5 服务名 off/on，在所有运行级别下设置，直接省略–level
监控

netstat [选项]：查看网络服务情况

选项：

-an：按照一定顺序排列输出
-p：显示那个进程在调用

10. 其他

  • ntsysv：查看系统自启动服务

  • echo [选项] [输出内容]：输出内容到控制台，常和重定向配合使用

  • **> / >>**：输出重定向；将命令执行结果输出（将原来文件的内容覆盖）或追加到指定文件中

    echo hello > a.txt ；将 hello 写到 a.txt 中
    ls -l > a.txt：将ls -l显示的内容覆盖写入到a.txt里面，如文件不存在，则创建文件
    ls -l >> a.txt：将ls -l显示的内容追加写入到a.txt里面

  • | 管道命令：可以将一个命令的输出作为另一个命令的输入

例：cat -n hello.txt | more，以cat命令打开hello.txt，将内容以分页带有行号显示
