写在前面：本文主要记录 MongoDB使用基础 的相关基础知识

1. 介绍

MongoDB：是一个基于分布式文件存储的、开源、高性能、无模式的 Nosql数据库；支持的数据结构非常松散，可以通过 BSON（类似于 JSON 的 格式，二进制的 JSON数据） 来存储数据；既可以存储比较复杂的数据类型，又相当的灵活；MongoDB 中的记录是一个文档，由字段和值对（fifield:value）组成；MongoDB是最像关系型数据库（MySQL）的非关系型数据库；官网，官方文档，官方中文文档，中文社区

BSON：Binary Serialized Document Format，是一种类json的一种二进制形式的存储格式，简称Binary JSON；支持内嵌的文档对象和数组对象，具备丰富的数据类型

1.1 应用场景

三高需求：高可用、高性能、高并发

High performance：高并发

Huge Storage：高性能

High Scalability & High Availability：高可扩展性和高可用

1.2 区别

SQL术语/概念 MongoDB术语/概念 解释说明
database database 数据库
table collection 数据库表/集合
row document 数据记录行/文档
column fifield 数据字段/域
index index 索引
table joins 嵌入文档 表连接；MongoDB不支持表连接；通过嵌入式文档来替代多表连接
primary key primary key 主键；MongoDB自动将_id字段设置为主键

1.3 特点

高性能：MongoDB提供高性能的数据持久性；对嵌入式数据模型的支持减少了数据库系统上的I/O活动；索引支持更快的查询，并且可以包含来自嵌入式文档和数组的键（文本索引解决搜索的需求、TTL索引解决历史数据自动过期的需求、地理位置索引可用于构建各种 O2O 应用）；mmapv1、wiredtiger、mongorocks（rocksdb）、in-memory 等多引擎支持满足多种场景，Gridfs解决文件存储

高可用性：MongoDB的复制工具称为副本集（replica set），提供自动故障转移和数据冗余

高扩展性：MongoDB提供水平可扩展性；分片将数据分布在一组集群的机器上；从3.4开始，MongoDB支持基于片键创建数据区域，在一个平衡的集群中，MongoDB将一个区域所覆盖的读写只定向到指定区域内的指定片

丰富的查询支持：MongoDB支持丰富的查询语言，支持读和写操作(CRUD)，比如数据聚合、文本搜索和地理空间查询等

其他特点：无模式（动态模式）、灵活的文档模型

2. 安装

2.1 Windows安装

下载地址，选择合适的版本进行下载安装

  1. 将压缩包解压到一个目标目录中（自定义）

  2. 在解压目录中，手动建立一个目录用于存放数据文件，data/db；推荐与 bin 同级

  3. 启动服务：默认端口是27017

    • 命令启动：在 bin 目录中打开 cmd命令行提示符，输入命令：mongod –dbpath=..\data\db

    • 配置文件启动：在解压目录中新建 config 文件夹（与 bin 同级），该文件夹中新建配置文件 mongod.conf；更多配置参考：配置文件参考地址

      storage:
    # 设置数据文件路径，D:\02_Server\DBServer\mongodb-win32-x86_64-2008plus-ssl-4.0.1，需要修改为自己的路径
    dbPath: D:\02_Server\DBServer\mongodb-win32-x86_64-2008plus-ssl-4.0.1\data

启动：mongod --config ../config/mongod.conf （在 bin 目录中打开 cmd命令行提示符，运行）

  4. 远程连接：在命令行提示符运行：mongo –host=127.0.0.1 –port=27017 完成登录，可通过 mongo –help 查看帮助

2.2 安装图形化界面-Compass

下载地址，选择合适的版本进行下载安装；如果是下载安装版，则按照步骤安装；如果是下载加压缩版，直接解压，执行里面的 MongoDBCompassCommunity.exe 文件即可

连接：在打开的界面中，输入主机地址、端口等相关信息，点击连接；完成与本地 MongoDB 连接，以后操作可在 Compass 中进行

2.3 Linux安装

  1. 下载地址，官网下载压缩包 mongod-linux-x86_64-x.x.x.tgz

  2. 上传压缩包到Linux中，解压到当前目录：tar -xvf mongodb-linux-x86_64-x.x.x.tgz

  3. 移动解压后的文件夹到指定的目录：mv mongodb-linux-x86_64-x.x.x /usr/local/mongodb

  4. 新建目录：用来存储数据和日志

# 日志存储目录

  5. 新建配置文件：vi /mongodb/single/mongod.conf

    # 内容如下：

    # The path of the log file to which mongod or mongos should send all diagnostic logging information

    logAppend: true

    # mongod实例存储其数据的目录。storage.dbPath设置仅适用于mongod。
    # The directory where the mongod instance stores its data.Default Value is "/data/db".
    dbPath: "/mongodb/single/data/db"
    journal:
        # 启用或禁用持久性日志以确保数据文件保持有效和可恢复。
        enabled: true
processManagement:
    # 启用在后台运行mongos或mongod进程的守护进程模式。
    fork: true
net:
    # 服务实例绑定的IP，默认是localhost
    bindIp: localhost,192.168.0.2
    # bindIp
    # 绑定的端口，默认是27017
    port: 27017

  6. 启动MongoDB服务

    /usr/local/mongodb/bin/mongodmongod --dbpath /usr/local/var/mongodb --logpath /usr/local/var/log/mongodb/mongo.log --fork
# --fork 指定后台运行
# 若要使用账号密码服务 需要在启动时加 --auth

/usr/local/mongodb/bin/mongod --config /usr/local/etc/mongod.conf
# /usr/local/mongodb/bin/mongod 为 mongod 脚本所在位置，可在 /usr/local/mongodb/bin 目录下将命令简化为 mongod
# 如果启动失败，检查配置文件

  7. 使用mongo命令和compass工具来连接测试登录

    mongo   # 默认无密码登录

# 密码登录
method-1：
mongo 127.0.0.1/admin -u admin -p 123456

method-2：
1. mongo进入shell
2. use admin
3. db.auth('admin','123456')

# 如果远程连接不上，需要配置防火墙放行，或直接关闭linux防火墙

systemctl stop firewalld

# 开机禁止启动防火墙
systemctl disable firewalld

  8. 停止关闭服务

3. 使用

3.1 数据类型

数据类型 描述 举例
字符串 UTF-8字符串都可表示为字符串类型的数据 {“x” : “foobar”}
对象id 对象id是文档的12字节的唯一 ID {“X” :ObjectId() }
布尔值 真或者假：true或者false {“x”:true}+
数组 值的集合或者列表可以表示成数组 {“x” ： [“a”, “b”, “c”]}
32位整数 类型不可用；JavaScript仅支持64位浮点数，所以32位整数会被自动转换 shell是不支持该类型的，shell中默认会转换成64位浮点数
64位整数 不支持；shell会使用一个特殊的内嵌文档来显示64位整数 shell是不支持该类型的，shell中默认会转换成64位浮点数
64位浮点数 shell中的数字就是这一种类型 {“x”：3.14159，”y”：3}
null 表示空值或者未定义的对象 {“x”:null}
undefifined 文档中也可以使用未定义类型 {“x”:undefifined}
符号 shell不支持，shell会将数据库中的符号类型的数据自动转换成字符串
正则表达式 文档中可以包含正则表达式，采用JavaScript的正则表达式语法 {“x” ： /foobar/i}
代码 文档中还可以包含JavaScript代码 {“x” ： function() { /* …… */ }}
二进制数据 二进制数据可以由任意字节的串组成，不过shell中无法使用
最大值/最小值 BSON包括一个特殊类型，表示可能的最大值；shell中没有这个类型

3.2 数据库操作

数据库命名规范：

  • 不能是空字符串（””)
  • 不得含有’ ‘（空格)、.、$、/、\和\0 (空字符)
  • 应全部小写
  • 最多64字节

保留数据库：

  • admin： 类似于 root数据库；数据库内新增一个用户，用户会自动继承所有数据库的权限；一些特定的服务器端命令也只能从这个数据库运行，比如列出所有的数据库或者关闭服务器

  • local：这个数据库永远不会被复制，用来存储限于本地单台服务器的任意集合

  • confifig：当Mongo用于分片设置时，confifig数据库在内部使用，用于保存分片的相关信息

选择或创建数据库：use db_name ；如果数据库不存在则自动创建，存在则切换当前数据库，默认的数据库为 test
显示当前数据库：db

查看所有数据库：show databases / dbs

删除数据库：db.dropDatabase() ；主要用来删除已经持久化的数据库

3.3 集合操作

集合：类似于关系型数据库中的表；既可以显示创建，也可以隐式创建；在 MongoDB 中，集合只有在内容插入后才会创建；即：创建集合(数据表)后要再插入一个文档(记录)，集合才会真正创建

集合的命名规范：

  • 集合名不能是空字符串””
  • 集合名不能含有\0字符（空字符)，这个字符表示集合名的结尾
  • 集合名不能以”system.”开头，这是为系统集合保留的前缀
  • 用户创建的集合名字不能含有保留字符 $：只有访问系统创建的集合时，才能在名字里出现$；某些系统生成的集合中包含该字符，会导致有些驱动程序支持在集合名里面包含保留字符
显示创建集合：db.createCollection(collection_name)

查看当前库中的表：show collections / tables

删除集合：成功删除选定集合，drop() 方法返回 true
db.collection_name.drop()

3.4 文档操作

文档：document，类似于关系型数据库的记录；数据结构和 JSON 基本一样，在集合中以 BSON 格式存储

文档键命名规范：

  • 键不能含有\0 (空字符)，这个字符用来表示键的结尾
  • .和$有特别的意义，只有在特定环境下才能使用
  • 以下划线”_”开头的键是保留的(非严格要求)

注意：

  1. 文档中的键/值对是有序的
  2. 文档中的值不仅可以是在双引号里面的字符串，还可以是其他几种数据类型（甚至可以是整个嵌入的文档)
  3. MongoDB区分类型和大小写
  4. MongoDB的文档不能有重复的键
  5. 文档的键是字符串；除了少数例外情况，键可以使用任意UTF-8字符

3.4.3 文档修改

db.collection_name.update(
    <query>,
    <update>,
    {
        upsert: <boolean>,
        multi: <boolean>,
        writeConcern: <document>,
        collation: <document>,
        arrayFilters: [ <filterdocument1>, ... ],
        hint: <document|string> // Available starting in MongoDB 4.2
    }
)
Parameter Type Description
query document 更新的选择条件：类似sql update 查询内 where 后面的 condition，可以使用与 fifind() 方法中相同的查询选择器；在3.0版中：当使用 upsert:true 执行 update() 时，如果查询使用点表示法在_id字段上指定条件，则MongoDB将拒绝插入新文档
update document or pipeline 要应用的修改；该值可以是：包含更新运算符表达式的文档，或仅包含：对的替换文档
upsert boolean 可选参数；设置为true，则在没有与查询条件匹配的文档时创建新文档；默认值为false，如果找不到匹配项，则不会插入新文档
multi boolean 可选参数；设置为true，则更新符合查询条件的多个文档；默认值为false，则更新一个文档
writeConcern document 可选参数；表示写问题的文档，抛出异常的级别
collation document 可选参数；指定要用于操作的校对规则；校对规则允许用户为字符串比较指定特定于语言的规则，具有以下语法：
校对规则:{
区域设置:,
caseLevel:,
caseFirst:,
强度:,
numericordering:,
替代:,
最大变量:,
向后:
}
指定校对规则时，区域设置字段是必需的；所有其他校对规则字段都是可选；如果未指定校对规则，集合具有默认校对规则（请参见db.createCollection()），则该操作将使用为集合指定的校对规则；如果没有为集合或操作指定校对规则，MongoDB将使用以前版本中使用的简单二进制比较进行字符串比较；不能为一个操作指定多个校对规则；例如：不能为每个字段指定不同的校对规则，或者如果使用排序执行查找，则不能将一个校对规则用于查找，另一个校对规则用于排序
arrayFilters array 可选参数；一个筛选文档数组，用于确定要为数组字段上的更新操作修改的数组元素
hint document or string 可选参数；指定用于支持查询谓词的索引的文档或字符串，该选项可以采用索引规范文档或索引名称字符串；如果指定的索引不存在，则说明操作错误

列修改：

$unset：删除列
$rename：重命名列

覆盖修改：文档内除了likenum字段其它字段都变为空

db.comment.update({_id:"1"},{likenum:NumberInt(1001)})  # 修改_id为1的记录，点赞量为1001

局部修改：使用修改器$set来实现，仅修改需要修改的字段

db.comment.update({_id:"2"},{$set:{likenum:NumberInt(889)}})  # 修改_id为2的记录，浏览量为889

批量修改：默认只修改第一条数据，通过 multi:true 实现批量修改

// 默认只修改第一条数据
db.comment.update({userid:"1003"},{$set:{nickname:"凯撒大帝"}})

// 修改所有符合条件的数据
db.comment.update({userid:"1003"},{$set:{nickname:"凯撒大帝"}},{multi:true})

运算修改：实现对某列值在原有值的基础上进行增加或减少，使用 $inc 运算符来实现

db.comment.update({_id:"3"},{$inc:{likenum:NumberInt(1)}})

3.4.4 文档查询

3.4.4.1 普通查询

db.collection_name.find(<query>, [projection])
Parameter Type Description
query document 可选参数；使用查询运算符指定选择筛选器，若要返回集合中的所有文档，请省略此参数或传递空文档( {} )
projection document 可选参数；指定要在与查询筛选器匹配的文档中返回的字段（投影），若要返回匹配文档中的所有字段，省略此参数 
db.comment.find()        # 默认查询所有列
db.comment.find({})

db.comment.find({userid:'1003'})     # 在find()中添加 json格式参数，实现条件查询
db.comment.findOne({userid:'1003'})  # 只返回一条符合条件的记录
db.comment.find({userid:"1003"},{userid:1,nickname:1})  # 查询结果只返回 _id、userid、nickname；默认会返回 _id
db.comment.find({userid:"1003"},{userid:1,nickname:1,_id:0})  # 查询结果只返回 userid、nickname

统计查询：使用 count() 方法

db.collection_name.count(query, options)

query：查询选择条件
options：可选参数，用于修改计数的额外选项

db.comment.count()  # 统计所有记录数，默认返回符合条件的全部记录条数
db.comment.count({userid:"1003"})  # 统计userid为1003的记录条数

分页查询：使用 limit() 方法来读取指定数量的数据，使用 skip() 方法来跳过指定数量的数据

db.collection_name.find().limit(NUMBER).skip(NUMBER)

limit：返回结果(TopN)，默认值20
skip：跳过的记录条数，默认值是0

db.comment.find().skip(2).limit(2)

排序查询：使用 sort() 方法对数据进行排序，使用 1(升序) 和 -1(降序) 来指定排序方式

db.collection_name.find().sort({KEY:1})

db.comment.find().sort({userid:-1,likenum:1})  # 对userid降序排列，并对点赞量进行升序排列
  • skip(), limilt(), sort() 三个一起执行时，执行的顺序是 **sort() -> skip() -> limit()**，和命令编写顺序无关

比较查询：gt、lt、gte、lte、ne

db.collection_name.find({"field_name":{$gt:value}})   # 大于: field > value
db.collection_name.find({"field_name":{$lt:value}})   # 小于: field < value
db.collection_name.find({"field_name":{$gte:value}})  # 大于等于: field >= value
db.collection_name.find({"field_name":{$lte:value}})  # 小于等于: field <= value
db.collection_name.find({"field_name":{$ne:value}})   # 不等于: field != value

db.comment.find({likenum:{$gt:NumberInt(700)}})  # 查询评论点赞数量大于700的记录

包含查询：使用 $in操作符 和 $nin操作符实现包含不包含

db.collection_name.find({"field_name":{$in:[value]}})

db.comment.find({userid:{$in:["1003","1004"]}})    # 查询评论的集合中userid字段包含1003或1004的文档
db.comment.find({userid:{$nin:["1003","1004"]}})   # 查询评论集合中userid字段不包含1003和1004的文档

模糊查询：通过 正则表达式 的方式实现，更多内容可查看 正则表达式

db.collection_name.find({field_name:/正则表达式/})

db.comment.find({content:/开水/})    # 查询评论内容包含“开水”的所有文档
db.comment.find({content:/^专家/})   # 查询评论的内容中以“专家”开头

条件逻辑连接查询：通过 $and 、$or 操作符实现

$and:[ { },{ },{ } ]
$or:[ { },{ },{ } ]

db.comment.find({$and:[{likenum:{$gte:NumberInt(700)}},{likenum:{$lt:NumberInt(2000)}}]})  # 查询评论集合中likenum大于等于700 并且小于2000的文档

db.comment.find({$or:[ {userid:"1003"} ,{likenum:{$lt:1000} }]})  # 查询评论集合中userid为1003，或者点赞数小于1000的文档记录

执行计划：通常使用执行计划（解释计划、Explain Plan）来查看查询情况，分析查询性能

db.collection_name.find(query,options).explain(options)
# "stage" : "COLLSCAN"：表示全集合扫描，性能较差
# "stage" : "IXSCAN"：表示基于索引扫描，性能较好

db.comment.find({userid:"1013"}).explain()

3.4.4.2 聚合查询

db.collection_name.aggregate([
	{管道:{表达式}}
	....
])

常用管道：

  • $group：将集合中的文档分组，用于统计结果
  • $match：过滤数据，只要输出符合条件的文档
  • $sort：聚合数据进一步排序
  • $skip：跳过指定文档数
  • $limit：限制集合数据返回文档数

常用表达式：

  • $sum：总和；$sum:1同count表示统计
  • $avg：平均
  • $min：最小值
  • $max：最大值
# 统计点赞人数
db.comment.aggregate([
	{
		$group:{	
			_id:"$userid",
			res:{$sum:1}
	    }
	}
])

# 统计点赞总数
db.comment.aggregate([
	{
		$group:{	
			_id:"$likenum",
			res:{$sum:likenum}
	    }
	}
])

3.5 常用命令

选择切换数据库：use articledb
插入数据：db.comment.insert({bson数据})
查询所有数据：db.comment.find();
条件查询数据：db.comment.find({条件})
查询符合条件的第一条记录：db.comment.findOne({条件})
查询符合条件的前几条记录：db.comment.find({条件}).limit(条数)
查询符合条件的跳过的记录：db.comment.find({条件}).skip(条数)
修改数据：db.comment.update({条件},{修改后的数据}) 或db.comment.update({条件},{$set:{要修改部分的字段:数据})
修改数据并自增某字段值：db.comment.update({条件},{$inc:{自增的字段:步进值}})
删除数据：db.comment.remove({条件})
统计查询：db.comment.count({条件})
模糊查询：db.comment.find({字段名:/正则表达式/})
条件比较运算：db.comment.find({字段名:{$gt:值}})
包含查询：db.comment.find({字段名:{$in:[值1，值2]}})或db.comment.find({字段名:{$nin:[值1，值2]}})
条件连接查询：db.comment.find({$and:[{条件1},{条件2}]})或db.comment.find({$or:[{条件1},{条件2}]})

4. 索引

索引：支持在MongoDB中高效地执行查询，MongoDB索引使用B树数据结构中的 B-Tree；详细内容查看 索引官网

分类：

  • 单字段索引：Single Field Index，用户定义在文档的单个字段上的升序 / 降序索引；对于单个字段索引和排序操作，索引键的排序顺序并不重要，MongoDB支持在任何方向上遍历索引
  • 复合索引：Compound Index，支持用户在多个字段定义索引；复合索引中列出的字段顺序具有重要意义：复合索引由 { userid: 1, score: -1 } 组成，则索引首先按userid正序排序，然后在每个userid的值内，再在按score倒序排序
  • 地理空间索引：Geospatial Index，通过 平面几何的二维索引和球面几何的二维球面索引 实现对地理空间坐标数据的有效查询
  • 文本索引：Text Indexes，支持在集合中搜索字符串内容
  • 哈希索引：Hashed Indexes，支持基于散列的分片，对字段值的散列进行索引；索引在其范围内的值分布更加随机，但只支持相等匹配，不支持基于范围的查询

4.1 创建

db.collection_name.createIndex(keys, options)   # 在 3.0.0 版本前创建索引方法为 db.collection_name.ensureIndex()，以后的版本还能用，推荐使用createIndex()

keys：包含字段和值对的文档，其中字段是索引键，值描述该字段的索引类型
options：可选参数；包含一组控制索引创建的选项的文档

options 参数：

Parameter Type Description
background Boolean 建索引过程会阻塞其它数据库操作，background参数可指定以后台方式创建索引；默认值为 false
unique Boolean 是否建立唯一索引；默认值为 false
name string 索引名称；如未指定，MongoDB的通过连接索引的字段名和排序顺序生成一个索引名称
dropDups Boolean 在建立唯一索引时是否删除重复记录，3.0+版本已废弃；默认值为 false
sparse Boolean 对文档中不存在的字段数据不启用索引，如果设置为true，在索引字段中不会查询出不包含对应字段的文档；默认值为 false
expireAfterSeconds integer 设定集合的生存时间，指定一个以秒为单位的数值，完成 TTL设定
v index version 索引的版本号，默认的索引版本取决于mongod创建索引时运行的版本
weights document 索引权重值，表示该索引相对于其他索引字段的得分权重；数值在 1 到 99999 之间
default_language string 对于文本索引，决定停用词及词干和词器的规则的列表； 默认为英语
language_override string 对于文本索引，指定包含在文档中的字段名，语言覆盖默认的language，默认值为 language 
db.comment.createIndex({userid:1})  # 对 userid 字段建立升序索引
db.comment.createIndex({userid:1,nickname:-1})  # 对 userid 和 nickname 建立复合索引：userid升序，nickname降序
db.comment.createIndex({userid:1,nickname:-1},{name:"idx_comp_id_nn"})  # 指定索引名称

4.2 删除

# 删除单个索引
db.collection_name.dropIndex(idx_name)

db.comment.dropIndex({userid:1})   # 安装索引定义规则删除，也可以按照名称删除
# 删除所有索引
db.collection.dropIndexes()

4.3 查看

db.collection_name.getIndexes()  # 返回一个集合中的所有索引的数组；默认有 _id 唯一索引(值不能重复)，默认索引名称为 _id_，不能在_id字段上删除此索引
