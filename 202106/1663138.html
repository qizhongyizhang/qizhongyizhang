IDEA-安装使用

Java IDEA

写在前面：本文主要记录Java实用工具-IDEA安装使用的相关知识

1. 介绍

IDEA全称Intellij IDEA是JetBrains公司的产品，是Java的集成开发环境，支持只能代码助手、代码自动提示、重构、J2EE支持、Ant、JUnit、CVS整合、代码审查和超前的GUI设计，支持Java、Scala、Groovy等语言，支持主流的技术和框架
优势

  1. 强大的代码整合功能（Git、Maven、Spring）
  2. 提示功能的快速、便捷、范围广(数据库表名、列名)
  3. 好用的快捷键和代码版本
  4. 精准搜索
  5. 实时保存

2. 下载安装

  1. 卸载：安装前保证环境干净（卸载干净），初次安装无问题
    通过控制面板进行卸载，不保留缓存和设置，卸载结束之后重启电脑

  2. 下载：下载官网 ，选择合适的版本

  3. 安装：
    双击下载的exe文件，进入安装；点击下一步，选择安装路径，点击下一步；选择电脑对应的位数，选择.java，点击下一步；选择默认，点击Install，开始安装；安装结束，启动IDEA，跳过自行默认选择
    安装目录介绍

启动目录介绍

  1. 配置环境
    • 虚拟机选项
  2. 启动
    运行IDEA安装目录的bin目录下的idea64.exe文件（32位的系统则运行idea32.exe，也可以在桌面创建快捷方式启动）

创建项目：点击Create New Project

下一步：选择创建的项目类型和所需版本的JDK

下一步：选择创建项目的方式：按照模板创建(勾选，点击Next)和自定已创建(不勾选，直接点击Next)

下一步：定义项目的名称和存放位置

Finish

.idea和project_name.ini文件是IDEA特定的文件

3. 配置使用

3.1 Module介绍

为适应分布式部署的需要，一个Project可以有多个Module组成，如：core Module、Web Module、plugin Module、solr Module等，彼此相互依赖
新建：右键点击项目名，选择New -> Module

下一步：选择创建的Module类型和所需版本的JDK

下一步：定义Module名称和存放目录

Finish：会在设置的位置出现对应的目录结构，进行编写
删除：右键点击模块名module_name，选择Open Module Settings，选中要删除的module_name，点击上方的减号，再右键点击模块名module_name，选择Delete

3.2 常用配置

进入设置界面有三种方法：

设置页面介绍

3.2.1 Appearance & Behavior

Theme
下载官网
file -> import Settings或者Settings -> color scheme的scheme进行导入
找到下载目录，导入后重启

取消更新
依次点击 Appearance & Behavior -> System Settings -> Update，取消勾选Automatically check update for

3.2.2 Keymap

IDEA和Eclipse有不同，可以进行搜索

3.2.3 Version Control(版本控制)

以 Git 为例

  1. 安装 Git 客户端
  2. 依次点击 Settings -> Version Control -> Git
  3. 在右侧的Path to Git executable ：里面配置 Git的安装目录
  4. 依次点击 Settings -> Version Control -> GitHub，Auth Type 选择Password ，在右侧输入账号和密码，测试连接
  5. 依次点击 VCS -> Checkout from Version Control -> GitHub

3.2.4 Editor

General
Mouse

  • Change font size with Ctrl + Mouse Wheel：通过鼠标滚轮改变字体大小
    Auto Import
  • Insert imports on paste 选择all，勾选Add unambiguous imports on the fly和Optimize imports on the fly (for current project)：自动导入相关jar包
    Appearance
  • show line numbers ：显示行号
  • show method separators：两个方法之间加入横线进行区分
    Code Completion
  • Case sensitive completions 选择None：忽略大小写，进行提示
    Editor tabs
  • show tabs in single row取消勾选：多行显示类文件
    Postfix Completion：模板不可修改
    Other
  • Show quick documentation on mouse move，设置时间，单位毫秒：鼠标放在类名上，会显示相关信息
    Font
    Font(字体) ：Consolas；Size(字体大小)： 16；Line spacing(行间距)：1.0
    Color Scheme
    Language Defaults -> Comments -> Block(多行)/Doc(文档)/Line(单行) Comment -> Foreground：设置注释颜色
    Color Scheme Font：设置编辑区字体
    Console Font，勾选Use console font instead of the default：设置控制台字体
    Code Style
    Java -> Imports -> Class count to use import with ‘*‘ ：设置import *的个数
    File and Code Templates
    Include -> File Header ：添加文档注释
    File Encoding
    Global(系统)/Project(当前项目) Encoding：设置文件的编码信息
    Live Template：可以定制
    psvm ：public static void main(String [] args){}
    sout ：System.out.println(“”); soutp、soutm、soutv：最近的变量值、xxx.sout
    fori ：for(int i =0; ; i++) iter、itar
    list.for:list.fori、list.forr
    ifn :inn
    prsf ：private static final
    psf ：public static final; psfi、psfs

3.2.5 Build、Execution、Deployment

Build Tools
配置Maven

Maven是项目自动化构建工具，也可以进行依赖管理
Maven home dirtionary ：选择Maven本地安装目录
User Setting File ：settings.xml
Local repository：本地仓库

勾选 import Maven projects automatically
Automatically download 勾选 Source 和 Doucmentation：可以选择下载源码和官方文档
VM options ：-Xmx768m

Compiler
Build project automatically：自动编译当前项目，建议勾选
Compiler independent modules in parallel：并行对多个模块进行编译，建议勾选

Debugger
Transport 勾选 Shared memory ，可以节省内存空间

3.2.6 其他

注释模板

  • 类注释模板设置：IDEA –> Settings… –> Editor –> File and Code Templates –> Includes –>File Header，在里面输入注释模板
  • 方法注释模板设置：IDEA –> Settings… –> Editor –> Live Templates –> 点击+号 –>Template Group…(添加模板组，随便取名)； 添加了模板组后再次点击+号，这次选择Live Template(添加注释模板）； Abbreviation: 该处填入* （此处一定要填写*,IDEA生成注释的默认方式是：/*+模板名+快捷键）

省电模式
File -> Power Save Mode：关闭代码检查和代码提示的功能

显示模式
右键点击类名 -> Split Vertically/Horizontally：横/竖分屏显示

生成JavaDoc
依次点击 Tools -> Generate JavaDoc， 可以选择生成的范围和存储目录，保存格式设置为中文：zh_CN ，Other command line arguments：-encoding UTF-8 -charset UTF-8

清除缓存
依次点击 File -> Invalidate Caches / Restart

3.3 配置Tomcat

运行JavaEE动态Web需要配置Tomcat项目

  1. 依次点击 Run -> Configuration
  2. 点击弹窗左上方的 + 号，选择 Tomcat -> Local
  3. 进入配置页面，点击Server页面，给 Tomcat 命名，点击 Configure 指明Tomcat的安装路径
  4. 点击Deployment，进入部署页面，依次点击右侧 + 号 -> Artifact ，选择需要部署的项目，在 Application context 指明部署应用的名称，点击Apply
    结束时，先断开与服务器的连接，在执行关闭

3.4 连接数据库

  1. 依次点击右侧的 Database -> +号 -> Data Source，选择数据库类型，进入连接配置页面
  2. 配置连接IP、端口、数据库、用户和密码，配置完成后可以点击 Test Connection，进行连接

3.5 断点调试

step over：进入下一步，如果当前行断点是一个方法，则不进入当前方法体内
step into：进入下一步，如果当前行断点是一个方法，则进入当前方法体内
force step into：进入下一步，如果当前行断点是一个方法，则进入当前方法体内
step out：跳出
resume program：恢复程序运行，但如果该断点下面代码还有断点则停在下一个断点上
stop：停止
mute breakpoints：使所有断点失效
view breakpoints：查看所有断点
条件断点
查看值：Ctrl + U
